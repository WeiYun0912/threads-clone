# threads-clone

- 學到新的第三方登入套件 clerk，可以用來建立使用者資料，並且也有提供 OAuth 和 Org 管理，但可能因為地區的關係，clerk 網站響應的時間非常非常慢。
- 讀取資料的速度因為資料階層過深的關係，有時候會非常慢，用了很多 mongo 的 populate，對於複雜的資料結構，還是用 RDBMS 比較好。
- 學到了 Next 的資料夾結構管理。
- 學到了 shadcn/ui tailwind。 (重要)
- 程式碼非常多，需要時間消化。
